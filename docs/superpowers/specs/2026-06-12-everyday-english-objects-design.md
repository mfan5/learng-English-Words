# 生活物品英语学习网站设计

## 目标

为英语口语基础约雅思 5.0 的学习者提供一个轻量、没有考试压力的生活词汇网站。学习重点是把日常看到的物品与英文名词直接联系起来，并通过发音、常见搭配和简短例句帮助用户开口表达。

## 产品结构

网站采用“按场景自由学习”的单层结构：

1. 首页展示六个生活场景。
2. 用户点击场景进入对应词卡列表。
3. 用户逐张查看真实图片、英文、中文、音标、发音、搭配和例句。
4. 用户确认掌握后点击“学会了”。
5. 网站记录当天学过的卡片，并展示今日学习数量。

第一版不设置强制每日任务、考试分数、录音或机器语音评分。

## 内容范围

第一版包含约 120 个高频生活物品，每个场景约 20 个：

- 厨房 Kitchen
- 客厅 Living room
- 卧室 Bedroom
- 浴室 Bathroom
- 办公室 Office
- 街区 Neighborhood

词汇选择以日常使用频率和口语实用性为标准，避免学术词汇和考试导向内容。

## 首页

首页包括：

- 网站名称和简短学习引导。
- “今天已学 N 张卡片”统计。
- “全部场景”和“今日学过”两个视图。
- 六张使用真实场景照片的分类入口。
- 每个入口显示中英文场景名、代表词汇和词汇数量。

“今日学过”只展示用户在当前自然日主动标记为“学会了”的卡片。

## 场景学习页

场景页顶部包括：

- 返回全部场景。
- 场景中英文名称。
- 简短操作提示。
- 当前场景已学数量和进度。

每张词卡包括：

- 一张能清楚识别物品的真实照片。
- 英文单词。
- 中文释义。
- 国际音标。
- 发音播放按钮。
- 一个高频口语搭配。
- 一句简短、自然、适合初级学习者的生活例句。
- “学会了”按钮。

用户可再次点击“今日已学”取消标记，防止误操作。

## 发音

第一版使用浏览器提供的英文语音合成功能播放单词和例句，不依赖后端服务。优先选择可用的英语语音，并在不可用时保持页面其余功能正常。

发音按钮必须有清晰的文字提示和无障碍标签。

## 学习记录

学习记录保存在浏览器本地存储中，不要求登录：

- 记录词卡 ID 和标记日期。
- 日期按用户本地时区计算。
- 首页统计当前日期标记的卡片数量。
- 场景页统计当前日期在该场景标记的数量。
- “今日学过”视图只读取当天记录。
- 历史记录可保留，但第一版不提供复杂的数据统计页面。

刷新页面或关闭浏览器后，当天记录仍然存在。

## 视觉方向

- 使用真实照片，不采用插画作为词卡主体。
- 整体气质温和、清晰、生活化，不像考试软件。
- 主色使用深绿色，操作强调色使用温暖橙色。
- 保持较大的图片、英文单词和点击区域。
- 桌面端采用多列卡片，手机端自然收缩为单列。
- 避免积分、排行榜、连续签到等制造压力的元素。

## 技术方案

第一版采用无构建步骤的静态单页实现：

- HTML 提供语义结构。
- CSS 提供响应式布局和视觉样式。
- JavaScript 管理场景切换、筛选、发音和本地学习记录。
- 词汇数据集中存放在 JavaScript 数据结构中，由页面动态渲染。
- 所有场景图和词卡图片下载到项目本地资源目录，页面不依赖外部图片链接。
- 图片按词卡实际展示尺寸压缩，采用清晰但不过大的常规网页尺寸，以兼顾加载速度和辨识度。
- 图片加载失败时仍提供可读的替代状态。

不引入框架、数据库、账号系统或服务器端 API。

## 主要组件

- 顶部导航与今日统计
- 首页介绍区
- 场景分类网格
- 场景详情头部与进度
- 词卡网格
- 单张词卡
- 今日学过筛选视图
- 空状态和图片加载失败状态

## 交互与状态

- 点击场景卡进入场景详情。
- 点击返回按钮回到场景列表。
- 点击发音按钮播放当前英文。
- 点击“学会了”写入当天记录并更新所有统计。
- 再次点击取消当天记录。
- 点击“今日学过”查看当天标记的全部词卡。
- 使用浏览器前进、后退时，页面视图应保持可理解。

## 验收标准

- 六个场景均可进入，并包含约 20 张不同词卡。
- 每张词卡具有完整的中英信息、音标、搭配和例句。
- 发音按钮在支持语音合成的主流浏览器中可工作。
- “学会了”状态在刷新后仍保留。
- 今日总数、场景进度和今日筛选结果保持一致。
- 桌面和手机宽度下没有横向溢出或不可读内容。
- 图片失败不会导致词义和操作不可用。
- 断开网络后，本地图片仍可正常显示。
- 页面无需安装依赖即可本地打开或通过简单静态服务器使用。

## 明确不包含

- 用户登录和云同步
- 录音、语音识别和发音评分
- 强制每日十词
- 考试、排行榜、积分和连续签到
- 后台内容管理系统
- 多语言界面
